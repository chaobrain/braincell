# BrainCell v4 — 功能需求与约束说明书
> 本文档是纯描述性需求单，不包含任何代码结构或实现细节。
> 目的：作为独立设计任务的输入，让不同的人或 AI 在不受现有实现影响的情况下，
> 自由设计各模块的代码结构，最后按接口契约对接。

---

## 背景与目标

构建一个用于多房室神经元仿真的基于Jax的Python库。
用户可以描述神经元的几何形态、为不同区域分配离子通道和电缆属性、
施加电流刺激、运行仿真并记录电压等状态变量。

整个系统分为五个功能层，各层职责清晰，可独立设计和测试。

---

## 全局约束

- 数值计算底层：所有涉及数组运算的部分必须基于 **JAX**（`jax.numpy`）
- 状态管理：神经元状态变量（电压、门控变量等）的生命周期管理基于 **brainstate**
- 物理单位：所有物理量（长度、电压、电导等）必须携带单位，基于 **brainunit**
- 不可变优先：描述性数据结构（形态、网格、机制声明）应设计为不可变对象
- 无循环依赖：五个功能层之间的依赖关系是单向的（见各层说明）

---

## 第一层：形态描述（morph）

### 目标

提供一套数据结构，完整描述神经元的三维几何形态。
形态由若干"分支"（branch）组成，分支之间形成树状拓扑结构。
还没真正决定Branch是否需要携带拓扑连接信息，你可以自由发挥，以下的示范例子是没有仅供参考。

### 功能需求

**分支（Branch）**

- 一条分支代表一段神经突起，由若干连续的圆台段组成
- 每段有长度和两端半径（近端半径、远端半径）
- 可选地携带三维坐标信息
- 支持多种构建方式：
  - 从三维坐标点序列构建
  - 从段长 + 连续半径序列构建
  - 从段长 + 每段独立的近端/远端半径构建（允许分叉处半径不连续）
- 可携带名称和类型标签（如 soma、axon、basal_dendrite、apical_dendrite）
- 能够计算自身的几何属性：段数、总长度、各段侧面积、各段体积

**形态树（Morpho）**

- 持有所有分支及其拓扑关系（父子连接）
- 每条连接需记录：位置点在子父分支的哪个位置parent_x和child_x
- 支持多种构建方式：
  - 从标准文件格式读取：SWC、ASC（Neurolucida）、NeuroML2
  - 手动逐步构建：先建根分支，再逐条追加子分支
  - 手动批量构建：一次性提供所有分支和连接关系
  - **语法糖构建**：支持类似 `soma.dend = Branch(...)` 的属性赋值语法，
    自动推断父子关系，最终生成不可变形态树。
    这种方式应尽量简洁直观，降低手动构建的心智负担。
- 支持不可变操作（每次操作返回新对象，原对象不变）：
  - 追加一条子分支
  - 在某位置将一条分支一分为二
  - 提取以某分支为根的子树
- 支持查询：按名称或编号获取分支、获取某分支的所有子分支、获取到根的路径

**几何度量**

- 计算从某分支到根的路径长度
- 计算形态上任意两个位点之间的树上最短路径长度
- 计算整个形态的总表面积、总长度

### 约束

- 不依赖 JAX 或 brainstate（纯 Python + brainunit）
- 分支对象应可被多个形态树安全共享（不可变）
- 拓扑信息不存储在分支对象内部，由形态树统一管理

## 第二层：区域与位点选择（filter）

### 目标

提供一套表达式系统，让用户能够在形态树上灵活地描述"哪些区域"或"哪些位点"，
用于后续的通道分配、刺激施加等操作。

### 功能需求

**区域选择（Region）**

- 区域是形态上的一个连续或离散的空间范围，由若干分支的区间组成(branchid, prox, dist)
- 内置选择器：
  - 全部区域、空区域
  - 单条分支的指定区间
  - 按分支类型或名称筛选（如"所有 soma 类型的分支"）
  - 按半径范围筛选
  - 按到 soma 的树上距离范围筛选
  - 按到 soma 的欧氏距离范围筛选
  - 某分支的整个子树
- 支持集合运算：并集、交集、差集、补集

**位点选择（Locset）**

- 位点是形态上的离散点集合，每个点由（brainchid，x）表示
- 内置选择器：
  - 根分支的指定位置
  - 所有分叉点
  - 所有末梢
  - 某区域内每条分支的指定位置
  - 某区域内均匀采样 N 个点
  - 某区域内按固定步长采样

**求值**

- 选择器是惰性的：定义时不计算，只在需要时传入形态树求值
- 对同一形态树求值多个选择器时，应能共享中间计算结果（如树上距离），避免重复计算

### 约束

- 依赖第一层（形态描述）
- 不依赖 JAX 或 brainstate

## 第三层：可视化（vis）

### 目标

将形态树渲染为图形，支持 2D 和 3D，支持叠加区域高亮、位点标记和数值着色。

### 功能需求

- 2D 渲染：将形态投影到平面，按分支类型着色，基于 matplotlib
- 3D 渲染：三维展示形态，支持交互旋转，优先使用 pyvista，备选 plotly（适合 Jupyter）
- 叠加功能：
  - 高亮指定区域（RegionFilter 结果）
  - 标记指定位点（LocsetFilter 结果）
  - 按数值数组着色（如仿真结束时各节点的电压值）
- 后端自动选择：运行时检测可用库，按优先级选择（pyvista → plotly → 仅 matplotlib）
- 提供统一入口函数，用户无需关心后端细节

### 约束

- 依赖第一层（形态描述），可选依赖第二层（区域/位点选择）
- matplotlib 是必选依赖，pyvista 和 plotly 是可选依赖

## 第四层：机制声明（mech）

### 目标

提供一组纯数据容器，让用户声明要施加的物理机制，不含任何计算逻辑。

### 功能需求

**电缆属性**：描述一段膜的被动电学属性
- 静息电位（mV）
- 膜电容（uF/cm²）
- 轴向电阻率（ohm·cm）

**密度机制**：描述在某区域施加某种离子机制或者离子通道机制
- 指定离子类型（来自现有 `mech/ion/` 库，保持不动）
- 指定通道类型（来自现有 `mech/channel/` 库，保持不动）
- 可携带传给通道的额外参数

**点机制**：描述在某位点施加的点机制
- 突触机制 （来自现有 `mech/synapse/` 库，保持不动）
- 电耦合机制
- 电刺激机制
- 探针机制

### 约束

- 不依赖任何其他层
- 不依赖 JAX 或 brainstate
- 纯数据，不可变

## 第五层：细胞编译与执行（cell）

### 目标

将形态描述和机制声明编译为可执行的仿真对象，完成从"描述"到"计算"的桥接。

### 功能需求

**声明式配置接口**

- 用户以链式调用的方式配置细胞：
  - `paint(区域, 机制...)` — 为某区域施加电缆属性或通道密度
  - `place(位点, 机制...)` — 在某位点施加刺激或放置探针
- 配置过程不触发任何计算，只是积累声明

**离散化**

- 将连续的形态树离散化为有限个计算单元（CV，control volume）
- 支持可配置的离散化策略（如每条分支分为 N 个 CV）
- 离散化结果包含：
  - 每个 CV 的面积、长度、所属分支
  - 节点之间的连接关系
  - 用于轴向电流计算的矩阵结构（DHS 三对角矩阵）
  - 唯一节点总数（`n_nodes`），这是通道实例化的关键参数

**编译**

- 将声明式配置编译为可执行对象，具体包括：
  - 根据区域掩码，将通道密度（g_max）转化为每个节点的空间变化数组（区域外为 0）
- 以 `size = (n_neurons, n_nodes)` 实例化各通道对象（与现有 `mech/channel/` 库对接，无需修改通道代码）
  - 组装轴向电流计算所需的矩阵
  - 注册刺激和探针

**执行**

- 编译后的对象可直接运行仿真，指定仿真时长和时间步长
- 仿真的 ODE 为：`dV/dt = (轴向电流 + 各通道电流之和 + 刺激电流) / 膜电容`
- 轴向电流通过 DHS 三对角矩阵求解
- 时间积分使用现有 quad/ 积分器（保持不动）
- 返回仿真结果，包含时间轴和各探针的记录数据

**与底层的对接**

- 编译后的对象继承自 `HHTypedNeuron`（来自 `_base.py`，保持不动）
- 状态变量（电压等）通过 brainstate 的 `DiffEqState` 管理
- 所有数值计算通过 JAX 执行，支持 `jax.jit` 加速
- 设计上应支持 `jax.vmap` 批量并行仿真（不同参数的多个细胞同时运行）

### 约束

- 依赖全部上游四层
- 依赖现有 `_base.py`、`mech/channel/`、`mech/ion/`、`quad/`（均保持不动，不修改）
- 单位在此层边界剥离（brainunit → 纯数值传给 JAX）

## 各层依赖关系

```
morph（无外部依赖）
  └─ filter（依赖 morph）
  └─ vis（依赖 morph，可选 filter）
  └─ cell（依赖 morph + filter + mech + 现有库）

mech（无外部依赖）
  └─ cell（依赖 mech）

现有库（保持不动，不修改）：
  _base.py / mech/channel/ / mech/ion/ / quad/
  └─ cell（依赖这些库）
```

## 端到端使用场景（描述性）

1. 用户从 SWC 文件加载形态，或手动用语法糖逐步构建形态树
2. 用户调用可视化功能预览形态，确认结构正确
3. 用户声明配置：为全部区域设置电缆属性，为 soma 区域添加 HH 通道，在 soma 施加电流刺激，在 soma 放置电压探针
4. 用户触发编译，指定神经元数量（popsize）
5. 用户运行仿真，指定时长和步长
6. 用户从结果中读取电压记录，绘制动作电位波形
7. 可选：用 `jax.vmap` 批量扫描不同的 g_max 参数，并行运行多组仿真
